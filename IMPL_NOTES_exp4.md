# Exp 4 — GR as a retain-hack PROHIBITION: implementation notes

Goal: use gradient routing SOLELY to forbid the retain adapter from learning
hacks (not to learn a forget representation). 128 reinterpreted "routing" samples
+ 512 STANDARD coherence (`coherence_rh_mode=none`) per step. NO reward penalty
anywhere; the −1 retain multiplier acts on the RAW-reward advantage. Generation of
the 128 uses BOTH adapters (1,1).

## (a) Code changes (file:line)

**advantages.py**
- `retain_prohibition_masks(mode)` — `advantages.py:94`. Single source of truth.
  Returns `(routing_forget_fwd_scale, rgm_good, fgm_good, rgm_bad, fgm_bad)`:
  - a: `(1, −1, 1, −1, 1)` → triple `(1, −1, 1)` for all routing samples.
  - b: `(0, −1, 0, −1, 0)` → triple `(0, −1, 0)` (off-policy retain-only fwd).
  - c: `(1, −1, 1, −1, 3)` → good `(1, −1, 1)`, bad `(1, −1, 3)`.

**train.py**
- `__init__` signature: `retain_prohibition_mode="none"` — `train.py:561`.
- Validation + store — `train.py:739`. Stores `self._retain_prohibition_mode`
  (`None` when `"none"`). Fail-loud asserts when set: requires
  `gradient_routing_enabled`, `renormalization_mode=='balanced'`, **`split_moment`
  OFF**, and `routing_lambda==1.0`.
- Fused-path override — `train.py:4127`. `routing_fs = train_fs` by default (no
  behavior change); when `_retain_prohibition_mode` is set it asserts `not slow`
  and split-moment off, then DIRECTLY overrides
  `routing_fs, rgm_good, fgm_good, rgm_bad, fgm_bad` from
  `retain_prohibition_masks(...)`, bypassing `routing_grad_mask_weights`/κ. Logs
  `retain_prohibition/{routing_fs,rgm_good,fgm_bad}`.
- vals loop now uses `routing_fs` (not `train_fs`) for the routing samples'
  forward forget-scale — `train.py:4207,4209`. Coherence stays `(0,1,0)`.
- argparse `--retain_prohibition_mode {none,a,b,c}` — `train.py:5043`
  (confirmed in `--help`).
- Construction-site wiring `retain_prohibition_mode=args.retain_prohibition_mode`
  — `train.py:6099`. Also flows through `train_main(params)` (valid argparse dest;
  set via `setattr`).

**Chosen approach (point 1):** the dedicated, auditable `--retain_prohibition_mode
{a,b,c}` flag, resolving to explicit mask constants defined in ONE place
(`retain_prohibition_masks`). Preferred over a free-form `--routing_grad_masks`
string because each mode also needs a routing forward forget-scale and (for b)
off-policy semantics; a single named function keeps all three configs explicit,
testable, and impossible to mis-specify per-run.

**Single source / adapter-type isolation:** the override only touches the
per-token `(forget_fwd, retain_gm, forget_gm)` triple, which both DualMLP and
DualLoRA consume identically via `gradient_routing._fused_decouple` — no
adapter-type-specific logic added outside `gradient_routing.py`.

## (b) Mode (b): old_logps at (1,1), update forward at (1,0)

Confirmed, and it required **no new old_logps machinery** — the existing routing
old_logps already equals the generation policy:
- Generation: routing samples are generated by vLLM at `(1, rollout_forget_scale)`
  = `(1, 1)` (`rollout_forget_scale_mode='fixed'` → 1.0; `forget_scale_clamp`=1.0).
- old_logps: the first pass computes `old_per_token_logps` at `(1,1)` (fast IS:
  vLLM logprobs from the (1,1) rollout; slow IS: HF forward at `(1, train_fs=1)`).
  **Only the coherence slice is recomputed at `(1,0)`** (`train.py:2697-2727`); the
  routing slice is never recomputed, so routing old_logps stays at `(1,1)`.
- Update forward: mode (b) sets `routing_fs = 0` ⇒ `forget_fwd = 0` on the routing
  tokens ⇒ the update forward is retain-only `(1,0)`. The IS ratio
  `exp(logp_(1,0) − logp_(1,1))` handles the genuine off-policy mismatch, exactly
  mirroring how coherence is off-policy (generate/old_logps at one scale, update
  forward at another). Verified the coherence recompute does not touch the routing
  slice.

## (d) −1 acts on RAW reward; no double-negation

- No reward penalty is applied to the 128: `coherence_rh_mode='none'`, no
  `reward_penalty_baseline`, and the routing-sample advantage is the balanced
  routing-group advantage `_baseline_nonflagged_var_all(raw_rewards, ...)`
  (`advantages.py:475`) — computed from RAW rewards.
- The sign flip lives ONLY in `retain_grad_mask = −1`. We do NOT also negate the
  advantage. Negating `retain_grad_mask` is mathematically equivalent to negating
  the retain advantage; exactly one of the two is applied (the mask). The forward
  forget-scale and forget grad-mask are non-negative; only the retain grad-mask is
  −1.

## (f) Uncertainties / flags for the researcher

1. **Batch-param spec discrepancy (resolved deliberately).** The task said
   `rollout_batch_size=640`. The code treats `rollout_batch_size` as the ROUTING
   count and ADDS coherence on top (`total = rollout_batch_size + coh`; verified
   from a real run log: `rollout=512+coh128 optimizer=640`). To realize the
   stated M/N = 128/512 (128 routing + 512 coherence, total 640 = one optimizer
   step) the sweep uses `rollout_batch_size=128, coh_samples_per_rollout=512`.
   Using 640 would produce 640 routing samples, breaking the design. 128/512/640
   are all multiples of `num_generations=32`. (The EXPERIMENTS doc's "Mapping:
   rollout_batch_size = M+N" line is inconsistent with the code and should
   probably be corrected to "rollout_batch_size = M".)
2. **balanced + split_moment OFF** is a valid but less-traveled combination (the
   matrix balanced sweeps pair balanced WITH split_moment). At λ=1 the fused fast
   path runs with `pre_routing_cap=None` and plain AdamW; κ is still derived/
   geometry-asserted at init but unused for routing samples (we override the
   masks). The smoke run is the place to confirm nothing downstream silently
   assumes split-moment under balanced.
3. **Modes (a)/(b) keep detection on** (`rh_detector_recall=1.0`) but set good/bad
   masks identically, so the good/bad split is irrelevant — only the routing-group
   advantage uses `is_rh` (for the non-flagged baseline). If the researcher wants
   (a)/(b) to literally not run detection, that is a separate change; current
   behavior is observationally identical for the gradient.
4. **Warm-start data** (`warmstart_data/`, v2 jsonl for sort/topic) is assumed
   present at run time (shared base). Not exercised here (no GPU training run).
5. Did not run GPU training (per instructions); smoke runs above are unexecuted.
